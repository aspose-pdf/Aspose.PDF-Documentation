{{% alert color="primary" %}}

Выпущена новая версия Aspose.PDF для Java с автоматическим портированием, и теперь этот единый продукт поддерживает возможность создавать PDF-документы с нуля, а также предоставляет возможность управлять существующими PDF-документами.

{{% /alert %}}

## Двоичные файлы продукта

{{% alert color="primary" %}}

В первой версии релиза (v4.0.0) мы поставляем два jar-файла, т.е. **aspose.pdf-3.3.0.jdk15.jar** и **aspose.pdf-new-4.0.0.jar**.
{{% /alert %}}

- **aspose.pdf-3.3.0.jdk14.jar**

Этот jar файл имеет ту же версию продукта, что и в настоящее время доступна в модуле загрузки с заголовком [Aspose.PDF for java 3.3.0](http://www.aspose.com/community/files/72/java-components/aspose.pdf-for-java/entry417659.aspx). В дальнейшем мы будем называть эту версию релиза устаревшей версией Aspose.PDF for Java. В этом первом выпуске существующие пользователи Aspose.PDF for Java, возможно, не будут затронуты, поскольку они получают ту же версию релиза. Однако в скором времени мы удалим этот отдельный jar файл, и все классы и перечисления устаревшей версии Aspose.PDF for Java (до 4.x.x) станут доступны в пакете com.aspose.pdf.generator единого файла aspose.pdf-new-4.x.x.jar.
- **aspose.pdf-new-4.0.0.jar**
  Этот jar файл является новой автоматически перенесенной версией Aspose.PDF для .NET на платформу Java.
 Этот jar-файл содержит новую модель объекта документа (DOM) для создания, а также изменения существующих PDF-файлов, а также текущий Aspose.PDF.Kit для Java. Все классы и перечисления Aspose.PDF.Kit для Java собраны в пакете com.aspose.pdf.facades, и в третьем квартале 2013 года Aspose.PDF.Kit для Java будет прекращен. Поэтому мы рекомендуем нашим текущим клиентам Aspose.PDF.Kit для Java попробовать перейти на новый авто-портированный релиз.

Пожалуйста, ознакомьтесь со следующим постом в блоге, чтобы получить больше информации о структуре авто-портированного Aspose.PDF для Java.
