## Установка

### Установка с сайта Aspose

Этот пакет включает большой файл, который хранится в виде архива bzip2.

1. **Скачать** архив **Aspose.PDF for Rust via C++** с официального сайта Aspose.
   Последняя (самая свежая) версия указана вверху и загружается по умолчанию, когда вы нажимаете кнопку **Download**.
   Рекомендуется использовать эту последнюю версию. Скачайте предыдущую версию только в случае необходимости.
   Пример: `Aspose.PDF-for-Rust-via-CPP-25.6.zip`

   Формат имени файла архива: `Aspose.PDF-for-Rust-via-CPP-YY.M.zip`, где:
   - `YY` = последние две цифры года (например, `25` для 2025)
   - `M` = номер месяца от `1` к `12`

2. **Извлечь** архив в выбранный вами каталог `{path}` используя подходящий инструмент:

   - В Linux/macOS:

     ```bash
     unzip Aspose.PDF-for-Rust-via-CPP-YY.M.zip -d {path}
     ```

   - В Windows используйте встроенную функцию извлечения в Проводнике или любой инструмент для разархивирования (7-Zip, WinRAR).

3. **Добавьте** библиотеку как зависимость в ваш проект на Rust. Вы можете сделать это двумя способами:

   - **Использование командной строки:**

     ```bash
     cargo add asposepdf --path {path}/asposepdf
     ```

   - **Ручное редактирование `Cargo.toml`:**
     Откройте свой проект `Cargo.toml` и добавьте следующее под `[dependencies]`:

     ```toml
     [dependencies]
     asposepdf = { path = "{path}/asposepdf" }
     ```

4. **Соберите ваш проект** (`cargo build`). При первой сборке соответствующая динамическая библиотека для вашей платформы будет автоматически извлечена из `.bz2` архив в `lib` папка и привязана к вашему проекту.

**Примечания**

- Скрипт сборки пытается создать **символическую ссылку** на библиотеку в вашем каталоге вывода (например, `target/debug/`).
- Для **Linux и macOS** также необходимо следовать [Конфигурация времени выполнения](#runtime-configuration) раздел ниже, чтобы гарантировать, что исполняемый файл может найти библиотеку во время выполнения.
- Все `.bz2` архивы имеют соответствующие `.sha256` файлы контрольных сумм. Если контрольная сумма отсутствует или недействительна, сборка завершится с ошибкой.

### Установка с GitHub

Этот пакет содержит предварительно скомпилированные нативные библиотеки (`.dll`, `.so`, `.dylib`) которые хранятся в сжатом виде `.bz2` архивы внутри репозитория GitHub.

1. **Добавьте** библиотеку как зависимость в ваш проект на Rust. Вы можете сделать это двумя способами:

   - **Использование командной строки:**

     ```bash
     cargo add asposepdf --git https://github.com/aspose-pdf/aspose-pdf-rust-cpp.git
     ```

   - **Ручное редактирование `Cargo.toml`:**

     Откройте свой проект `Cargo.toml` и добавьте следующее под `[dependencies]`:

     ```toml
     [dependencies]
     asposepdf = { git = "https://github.com/aspose-pdf/aspose-pdf-rust-cpp.git" }
     ```

    > **Примечание:** Чтобы использовать конкретную версию выпуска, вы можете указать тег:
    >
    > ```toml
    > asposepdf = { git = \"https://github.com/aspose-pdf/aspose-pdf-rust-cpp.git", tag = "v1.26.1" }
    > ```

2. **Соберите ваш проект** (`cargo build`). При первой сборке соответствующая динамическая библиотека для вашей платформы будет автоматически извлечена из `.bz2` архив в `lib` папка и привязана к вашему проекту.

**Примечания**

- Вам не нужно вручную загружать или извлекать какие-либо файлы - всё включено в репозиторий GitHub.
- Скрипт сборки пытается создать **символическую ссылку** на библиотеку в вашем каталоге вывода (например, `target/debug/`).
- Для **Linux и macOS** также необходимо следовать [Конфигурация времени выполнения](#runtime-configuration) раздел ниже, чтобы гарантировать, что исполняемый файл может найти библиотеку во время выполнения.
- Все `.bz2` архивы соответствуют `.sha256` файлы контрольных сумм. Контрольная сумма проверяется перед распаковкой.
- Если проверка контрольной суммы не прошла или архив отсутствует, сборка завершится с подробной ошибкой.

### Установка из crates.io

Этот пакет доступен на [crates.io](https://crates.io/crates/asposepdf). 
Из-за ограничений размера опубликованный crate не включает нативные бинарные библиотеки ("`.dll`, `.so`, или `.dylib`).
Вы можете получить необходимые нативные библиотеки либо из официального архива дистрибутива (см) [Установка с сайта Aspose](#installation-from-aspose-website) или из репозитория GitHub (см [Установка с GitHub](#installation-from-github)).
Скрипт сборки будет находить, проверять и извлекать соответствующую нативную библиотеку из сжатого архива .bz2 во время процесса сборки.

1. **Добавьте** библиотеку как зависимость в ваш проект на Rust. Вы можете сделать это двумя способами:

   - **Использование командной строки:**

     ```bash
     cargo add asposepdf
     ```

   - **Ручное редактирование `Cargo.toml`:**

     Откройте свой проект `Cargo.toml` и добавьте следующее под `[dependencies]`:

     ```toml
     [dependencies]
     asposepdf = "1.26.1"
     ```

2. **Установите путь** к каталогу, содержащему нативные библиотеки, и загрузите необходимые файлы:

   - **Установите переменную окружения `ASPOSE_PDF_LIB_DIR`** указывает на папку, где вы разместите native `.bz2` архивы, их `.sha256` файлы контрольных сумм, и извлечённые собственные библиотеки (`.dll`, `.so`, `.dylib`, и также для Windows `.lib`):

     - На Linux/macOS:

       ```bash
       export ASPOSE_PDF_LIB_DIR=/path/to/lib
       ```

     - В Windows (Командная строка):

       ```cmd
       set ASPOSE_PDF_LIB_DIR=C:\path\to\lib
       ```

     - В Windows (PowerShell):
     
       ```powershell
       $env:ASPOSE_PDF_LIB_DIR = "C:\path\to\lib"
       ```

**Примечание к ASPOSE_PDF_LIB_DIR**

Этот `ASPOSE_PDF_LIB_DIR` переменная среды определяет рабочий каталог для скрипта сборки. Она используется **только во время компиляции**, чтобы найти, проверить и извлечь архивы нативных библиотек. Установка этой переменной **не** приводит к автоматическому добавлению каталога в путь поиска библиотек во время выполнения вашей системы (см [Конфигурация времени выполнения](#runtime-configuration)).

  - **Скачайте требуемое** `.bz2` архивы** и файлы контрольных сумм из репозитория GitHub [`lib/` папка](https://github.com/aspose-pdf/aspose-pdf-rust-cpp/tree/main/lib) и **поместите их** в папку, указанную в `ASPOSE_PDF_LIB_DIR`:

  - Для **Linux x64**, загрузить:
    - `libAsposePDFforRust_linux_amd64.so.bz2`
    - `libAsposePDFforRust_linux_amd64.so.bz2.sha256`

  - Для **macOS x86_64**, загрузите:
    - `libAsposePDFforRust_darwin_amd64.dylib.bz2`
    - `libAsposePDFforRust_darwin_amd64.dylib.bz2.sha256`

  - Для **macOS arm64**, загрузите:
    - `libAsposePDFforRust_darwin_arm64.dylib.bz2`
    - `libAsposePDFforRust_darwin_arm64.dylib.bz2.sha256`

  - Для **Windows x64**, загрузите:
    - `AsposePDFforRust_windows_amd64.dll.bz2`
    - `AsposePDFforRust_windows_amd64.dll.bz2.sha256`
    - `AsposePDFforRust_windows_amd64.lib` (нативная импортируемая библиотека, не сжата)

**Примечание:** Вам необходимо вручную загрузить эти файлы с GitHub и поместить их в каталог, указанный `ASPOSE_PDF_LIB_DIR`.  
Скрипт сборки автоматически распакует нативные библиотеки из `.bz2` архивы при первой сборке.

3. **Соберите** ваш проект (`cargo build`). При первой сборке нативная библиотека, соответствующая вашей платформе, будет автоматически извлечена из `.bz2` архивировать и связать с вашим проектом.

**Важно:** Для **Linux и macOS** вам также следует выполнить [Конфигурация времени выполнения](#runtime-configuration) раздел ниже, чтобы гарантировать, что исполняемый файл может найти библиотеку во время выполнения.

**Примечания**

- Этот `ASPOSE_PDF_LIB_DIR` переменная используется **только во время процесса сборки**, чтобы находить и извлекать архивы.
- Скрипт сборки пытается создать **символическую ссылку** на извлечённую библиотеку в вашем выходном каталоге (например, `target/debug/`).
- Вы должны предоставить папку, содержащую `.bz2` и `.sha256` файлы отдельно, поскольку эти бинарные архивы не распространяются через crates.io.
- Если требуемый архив отсутствует или контрольная сумма не совпадает, сборка завершится с подробной ошибкой.
- Те же бинарные файлы, которые использовались для установки через GitHub или сайт Aspose, могут быть повторно использованы здесь.
