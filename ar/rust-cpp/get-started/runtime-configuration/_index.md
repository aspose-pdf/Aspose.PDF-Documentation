> هذا مطلب قياسي عند استخدام المكتبات الديناميكية الأصلية في Rust.

في نظامي Linux و macOS، لا يقوم محمل النظام الديناميكي بالبحث تلقائيًا في دليل الملف التنفيذي ما لم يتم تكوين RPATH. لضمان تمكّن تطبيقك من العثور على مكتبة Aspose.PDF الأصلية في وقت التشغيل، تحتاج إلى تكوين **RPATH** (مسار البحث أثناء التشغيل).

يقوم برنامج البناء الخاص بنا باستخراج المكتبة ويحاول إنشاء رابط رمزي لها في دليل الإخراج الخاص بك (مثلاً، `target/debug/`). لتمكين القابل للتنفيذ من العثور عليه، اختر أحد الخيارات التالية:

## الخيار 1: تكوين على مستوى المشروع (مُوصى به)

إنشاء مجلد باسم `.cargo` في دليل الجذر لمشروعك (إذا لم يكن موجودًا) وأنشئ ملفًا باسم `config.toml` في داخله:

```toml
[target.'cfg(target_os = "linux")']
rustflags = ["-C", "link-arg=-Wl,-rpath,$ORIGIN"]

[target.'cfg(target_os = "macos")']
rustflags = ["-C", "link-arg=-Wl,-rpath,@loader_path"]
```

## الخيار 2: متغيّر البيئة RUSTFLAGS

ابنِ مشروعك باستخدام العلامة التالية:

```bash
# Linux
RUSTFLAGS="-C link-arg=-Wl,-rpath,\$ORIGIN" cargo build

# macOS
RUSTFLAGS="-C link-arg=-Wl,-rpath,@loader_path" cargo build
```

## الخيار 3: التثبيت على مستوى النظام (غير موصى به للتطوير)

إذا كنت تفضّل تثبيت المكتبة بشكل عالمي:

* **Linux:** انسخ `.so` ملف إلى `/usr/local/lib` و تشغيل `sudo ldconfig`.
* **macOS:** انسخ `.dylib` ملف إلى `/usr/local/lib`.

> **Windows**
> عادةً لا يلزم اتخاذ أي إجراء لأن المكتبة موجودة في نفس المجلد كما `.exe`. بدلاً من ذلك، يمكنك إضافة الدليل الذي يحتوي على `.dll` إلى نظامك `PATH` متغير البيئة.
