## التثبيت

### التثبيت من موقع Aspose

تتضمن هذه الحزمة ملفًا كبيرًا مخزنًا كأرشيف bzip2.

1. **تحميل** الأرشيف **Aspose.PDF for Rust عبر C++** من الموقع الرسمي لـ Aspose.
   الإصدار الأحدث (الأكثر حداثة) مدرج في الأعلى ويتم تنزيله تلقائيًا عند النقر على زر **Download**.
   يوصى باستخدام هذه النسخة الأخيرة. قم بتنزيل نسخة سابقة فقط إذا لزم الأمر.
   مثال: `Aspose.PDF-for-Rust-via-CPP-25.6.zip`

   تنسيق اسم ملف الأرشيف هو: `Aspose.PDF-for-Rust-via-CPP-YY.M.zip`, أين:
   - `YY` = آخر رقمين من السنة (مثالًا، `25` لـ 2025)
   - `M` = عدد الشهر من `1` إلى `12`

2. **استخراج** الأرشيف إلى الدليل المختار `{path}` باستخدام أداة مناسبة:

   - على Linux/macOS:

     ```bash
     unzip Aspose.PDF-for-Rust-via-CPP-YY.M.zip -d {path}
     ```

   - في نظام Windows، استخدم استخراج المستكشف المدمج أو أي أداة فك ضغط (7-Zip، WinRAR).

3. **أضف** المكتبة كاعتماد في مشروع Rust الخاص بك. يمكنك القيام بذلك بطريقتين:

   - **استخدام سطر الأوامر:**

     ```bash
     cargo add asposepdf --path {path}/asposepdf
     ```

   - **تحرير يدوي** `Cargo.toml`:**
     افتح مشروعك `Cargo.toml` وأضف ما يلي تحت `[dependencies]`:

     ```toml
     [dependencies]
     asposepdf = { path = "{path}/asposepdf" }
     ```

4. **بناء مشروعك** (`cargo build`). في البناء الأول، سيتم استخراج المكتبة الديناميكية المناسبة لمنصتك تلقائيًا من الـ `.bz2` أرشيف في الـ `lib` المجلد وربطه بمشروعك.

**ملاحظات**

- تقوم نصّ البناء بمحاولة إنشاء **رابط رمزي** إلى المكتبة في دليل الإخراج الخاص بك (على سبيل المثال، `target/debug/`).
- لكل **Linux و macOS**، يجب عليك أيضًا اتباع [تكوين وقت التشغيل](#runtime-configuration) القسم أدناه لضمان أن القابل للتنفيذ يمكنه العثور على المكتبة أثناء وقت التشغيل.
- الكل `.bz2` الأرشيفات لها ما يطابقها `.sha256` ملفات checksum. إذا كان checksum مفقودًا أو غير صالح، سيفشل عملية البناء.

### التثبيت من GitHub

تتضمن هذه الحزمة مكتبات أصلية مُجمعة مسبقًا (`.dll`, `.so`, `.dylib`) التي يتم تخزينها مضغوطة `.bz2` الأرشيفات داخل مستودع GitHub.

1. **أضف** المكتبة كاعتماد في مشروع Rust الخاص بك. يمكنك القيام بذلك بطريقتين:

   - **استخدام سطر الأوامر:**

     ```bash
     cargo add asposepdf --git https://github.com/aspose-pdf/aspose-pdf-rust-cpp.git
     ```

   - **تحرير يدوي `Cargo.toml`:**

     افتح مشروعك `Cargo.toml` وأضف ما يلي تحت `[dependencies]`:

     ```toml
     [dependencies]
     asposepdf = { git = "https://github.com/aspose-pdf/aspose-pdf-rust-cpp.git" }
     ```

    > **ملاحظة:** لاستخدام نسخة إصدار محددة، يمكنك تحديد علامة:
    >
    > ```toml
> asposepdf = { git = "https://github.com/aspose-pdf/aspose-pdf-rust-cpp.git", tag = "v1.26.1" }
    > ```

2. **بناء مشروعك** (`cargo build`). في البناء الأول، سيتم استخراج المكتبة الديناميكية المناسبة لمنصتك تلقائيًا من الـ `.bz2` أرشيف في الـ `lib` المجلد وربطه بمشروعك.

**ملاحظات**

- ليس عليك تنزيل أي ملفات يدويًا أو استخراجها - كل شيء مدرج في مستودع GitHub.
- تقوم نصّ البناء بمحاولة إنشاء **رابط رمزي** إلى المكتبة في دليل الإخراج الخاص بك (على سبيل المثال، `target/debug/`).
- لكل **Linux و macOS**، يجب عليك أيضًا اتباع [تكوين وقت التشغيل](#runtime-configuration) القسم أدناه لضمان أن القابل للتنفيذ يمكنه العثور على المكتبة أثناء وقت التشغيل.
- الكل `.bz2` الأرشيفات لديها تطابق `.sha256` ملفات التجزئة. يتم التحقق من التجزئة قبل فك الضغط.
- إذا فشل التحقق من المجموع الاختباري أو كان الأرشيف مفقودًا، سيفشل البناء مع خطأ مفصل.

### التثبيت من crates.io

هذه الحزمة متاحة على [crates.io](https://crates.io/crates/asposepdf). 
نظرًا لقيود الحجم، لا يتضمن الصندوق المنشور مكتبات الثنائيات الأصلية (`.dll`, `.so`, أو `.dylib`).
يمكنك الحصول على المكتبات الأصلية المطلوبة إما من أرشيف التوزيع الرسمي (انظر [التثبيت من موقع Aspose](#installation-from-aspose-website)) أو من مستودع GitHub (انظر [التثبيت من GitHub](#installation-from-github)).
سيقوم سكريبت البناء بتحديد موقع المكتبة الأصلية المناسبة والتحقق منها واستخراجها من أرشيف مضغوط بامتداد .bz2 أثناء عملية البناء.

1. **أضف** المكتبة كاعتماد في مشروع Rust الخاص بك. يمكنك القيام بذلك بطريقتين:

   - **استخدام سطر الأوامر:**

     ```bash
     cargo add asposepdf
     ```

   - **تحرير يدوي** `Cargo.toml`:**

     افتح مشروعك `Cargo.toml` وأضف ما يلي تحت `[dependencies]`:

     ```toml
     [dependencies]
     asposepdf = "1.26.1"
     ```

2. **حدد المسار** إلى الدليل الذي يحتوي على المكتبات الأصلية وقم بتنزيل الملفات المطلوبة:

   - **ضبط متغير البيئة** `ASPOSE_PDF_LIB_DIR`** للإشارة إلى المجلد حيث ستضع الملفات الأصلية `.bz2` الأرشيفات، الخاصة بهم `.sha256` ملفات التحقق من المجموع، والمكتبات الأصلية المستخرجة (`.dll`, `.so`, `.dylib`، ولأنظمة Windows أيضًا `.lib`):

     - على Linux/macOS:

       ```bash
       export ASPOSE_PDF_LIB_DIR=/path/to/lib
       ```

     - على Windows (موجه الأوامر):

       ```cmd
       set ASPOSE_PDF_LIB_DIR=C:\path\to\lib
       ```

     - على Windows (PowerShell):
     
       ```powershell
       $env:ASPOSE_PDF_LIB_DIR = "C:\path\to\lib"
       ```

**ملاحظة حول ASPOSE_PDF_LIB_DIR**

ال `ASPOSE_PDF_LIB_DIR` متغيّر البيئة يحدد دليل العمل لسكريبت البنية. يُستخدم **فقط أثناء التجميع** لتحديد الموقع، والتحقق، واستخراج أرشيفات المكتبة الأصلية. ضبط هذا المتغيّر **ليس** يضيف الدليل تلقائيًا إلى مسار بحث مكتبة وقت تشغيل نظامك (انظر [تكوين وقت التشغيل](#runtime-configuration)).

  - **قم بتنزيل المطلوب** `.bz2` الأرشيفات** وملفات التحقق من المجموع من مستودع GitHub [`lib/` مجلد](https://github.com/aspose-pdf/aspose-pdf-rust-cpp/tree/main/lib) و **ضعها** في المجلد المحدد في `ASPOSE_PDF_LIB_DIR`:

  - لـ **Linux x64**، التحميل:
    - `libAsposePDFforRust_linux_amd64.so.bz2`
    - `libAsposePDFforRust_linux_amd64.so.bz2.sha256`

  - لـ **macOS x86_64**، قم بتنزيل:
    - `libAsposePDFforRust_darwin_amd64.dylib.bz2`
    - `libAsposePDFforRust_darwin_amd64.dylib.bz2.sha256`

  - لـ **macOS arm64**، قم بالتنزيل:
    - `libAsposePDFforRust_darwin_arm64.dylib.bz2`
    - `libAsposePDFforRust_darwin_arm64.dylib.bz2.sha256`

  - لِـ **Windows x64**، قم بالتنزيل:
    - `AsposePDFforRust_windows_amd64.dll.bz2`
    - `AsposePDFforRust_windows_amd64.dll.bz2.sha256`
    - `AsposePDFforRust_windows_amd64.lib` (مكتبة استيراد أصلية، غير مضغوطة)

**ملاحظة:** تحتاج إلى تنزيل هذه الملفات يدويًا من GitHub ووضعها في الدليل المشار إليه بـ `ASPOSE_PDF_LIB_DIR`.  
ستقوم سكريبت البناء تلقائيًا بفك حزم المكتبات الأصلية من `.bz2` الأرشيفات في أول بناء.

3. **بناء** مشروعك (`cargo build`). في أول بناء، سيتم استخراج المكتبة الأصلية المتطابقة مع نظامك الأساسي تلقائيًا من `.bz2` أرشفة وربطها بمشروعك.

**مهم:** بالنسبة لـ **Linux و macOS**، يجب عليك أيضًا اتباع [تكوين وقت التشغيل](#runtime-configuration) القسم أدناه لضمان أن القابل للتنفيذ يمكنه العثور على المكتبة أثناء وقت التشغيل.

**ملاحظات**

- ال `ASPOSE_PDF_LIB_DIR` المتغيّر يُستخدم **فقط أثناء عملية البناء** لتحديد واستخراج الأرشيفات.
- يحاول برنامج بناء السكريبت إنشاء **رابط رمزي** إلى المكتبة المستخرجة في دليل الإخراج الخاص بك (مثلاً، `target/debug/`).
- يجب عليك تقديم المجلد الذي يحتوي على `.bz2` و `.sha256` ملفات بشكل منفصل، لأن هذه الأرشيفات الثنائية لا تُوزَّع عبر crates.io.
- إذا كان الأرشيف المطلوب مفقودًا أو فشل المجموع الاختباري، سيفشل البناء مع خطأ مفصل.
- يمكن إعادة استخدام نفس ملفات الثنائي المستخدمة للتثبيت عبر GitHub أو موقع Aspose هنا.
