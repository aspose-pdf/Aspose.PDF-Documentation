## نظرة عامة

Aspose.PDF لبيثون عبر .NET، واجهة برمجة تطبيقات معالجة PDF تسمح للمطورين بالعمل مع مستندات PDF دون الحاجة إلى ميكروسوفت أوفيس® أو أدوبي أكروبات أوتوميشن. يمكن استخدام Aspose.PDF لـ Python لتطوير تطبيقات Python 32 بت و 64 بت لأنظمة تشغيل مختلفة (مثل Windows و Linux) حيث تم تثبيت Python 3.5 أو إصدار أحدث.

## متطلبات النظام لنظام التشغيل Target Linux

- مكتبات وقت تشغيل GCC-6 (أو لاحقًا).

- تبعيات وقت تشغيل .NET Core. تثبيت .NET Core Runtime نفسه غير مطلوب.

- بالنسبة لبيثون 3.5-3.7: هناك حاجة إلى بناء pymalloc لبيثون. يتم تمكين خيار إنشاء بايثون --with-pymalloc افتراضيًا. عادةً ما يتم تمييز بنية pymalloc لـ Python بلاحقة m في اسم الملف.

- مكتبة بايثون المشتركة لبيثون. يتم تعطيل خيار إنشاء Python --enable-shared افتراضيًا، ولا تحتوي بعض توزيعات Python على مكتبة libpython المشتركة. بالنسبة لبعض منصات لينكس، يمكن تثبيت مكتبة libpython المشتركة باستخدام مدير الحزم، على سبيل المثال: sudo apt-get install libpython3.7. المشكلة الشائعة هي أن مكتبة libpython مثبتة في موقع مختلف عن موقع النظام القياسي للمكتبات المشتركة. يمكن إصلاح المشكلة باستخدام خيارات إنشاء Python لتعيين مسارات مكتبة بديلة عند تجميع Python، أو إصلاحها عن طريق إنشاء رابط رمزي لملف مكتبة libpython في موقع النظام القياسي للمكتبات المشتركة. عادةً ما يكون اسم ملف مكتبة libpython المشتركة هو libpythonx.ym.so.1.0 لبيثون 3.5-3.7، أو libpythonX.y.so.1.0 لبيثون 3.8 أو أحدث (على سبيل المثال: libpython3.7m.so.1.0، libpython3.9.so.1.0).
